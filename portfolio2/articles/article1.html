第一号記事: GitHub Pages 構築記録

  • ホーム

記事内容

このページは、GitHub Pages を使用してウェブサイトを構築する過程を記録したものです。

構築の流れ

  1. GitHub アカウントを作成し、新しいリポジトリを作成しました。
  2. ローカル環境に必要なファイル（index.html, style.css, script.js）を作成しました。
  3. Git コマンドを使用してリポジトリを初期化し、リモートリポジトリに接続しました。
  4. ファイルを GitHub にプッシュし、GitHub Pages を有効化しました。

技術的なポイント

  • GitHub の Personal Access Token を使用して認証を行いました。
  • Git コマンド git pull origin main --rebase を使用してリモートリポジトリとローカルリポジトリを同期しました。
  • GitHub Pages の設定で main ブランチをルートフォルダとして指定しました。
