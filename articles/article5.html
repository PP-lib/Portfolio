生産性見える化アプリの概要と進行計画

執筆者: ユーザー名 | 公開日: 2024-12-19

概要

本アプリケーションは、生産データと勤務データを統合し、生産性の可視化やレポート作成を行うためのツールです。出退勤記録や生産重量を自動的に計算し、KPIを算出して視覚的に提供します。

プロジェクトの進行計画

以下のマイルストーンに基づき、開発を進めます。

  • プロジェクト準備 (1週間以内)
    • Docker環境の設定
    • リポジトリ構築と依存ライブラリのセットアップ
  • データ統合モジュール開発 (3週間以内)
    • 出退勤データと生産重量データの統合
    • Workerマスタデータとの整合性チェック
  • KPI計算モジュール開発 (5週間以内)
    • 労働時間と生産性の計算ロジックを実装
    • 工程ごとのKPI集計
  • 可視化モジュール開発 (6週間以内)
    • Matplotlibを用いたKPIグラフの生成
    • グラフのPNG出力機能
  • レポート作成および印刷 (7週間以内)
    • Excel形式のレポート生成
    • 自動印刷機能の実装

Docker設定の概要

# Dockerfile
FROM python:3.10
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "main.py"]

Docker Composeを使用して開発環境を統一化し、効率的なチーム開発を実現します。
