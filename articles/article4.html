GitHubとWebアプリ開発の効率的な管理方法

概要

本記事では、Webアプリ開発プロジェクトにおけるGitHubを活用した効率的な管理方法について解説します。 また、プロジェクト開始1日目の具体的なタスクやスケジュール管理の手法についても紹介します。

今日の作業内容

  • GitHubリポジトリのセットアップ
    • プロジェクト専用のリポジトリを作成し、README.mdを初期化。
    • ディレクトリ構造を設計し、適切に分類。
  • 仕様書の初版作成
    • プロジェクトの概要、主な機能、システム構成をMarkdown形式で記載。
  • GitHub Projectsの利用
    • タスクを「To Do」「In Progress」「Done」に分類。
    • 初期タスクとして「仕様書作成」「スケジュール設定」などを登録。
  • 開発環境のセットアップ
    • MacOS上でPython仮想環境を構築。
    • Flaskアプリの初期構造を作成し、動作確認。

手順：GitHubを活用した開発プロジェクト管理

本日の作業を効率的に進めるために以下の手順を実施しました。

  • リポジトリの初期化とREADME.mdの記載。
  • Markdown形式での仕様書作成。
  • GitHub Projectsでタスクの可視化と進捗管理。
  • Python仮想環境のセットアップと初期動作確認。
